# Изменения в логике парсинга EIS

## Проблема
Парсер работал нестабильно:
- Задвоение записей при повторении наименований лекарственных средств
- Сдвиг данных по столбцам (Торговое наименование, лек. форма, дозировка смещались вправо)
- Попадание неверной информации в ячейки
- Пропуск позиций при наличии одинаковых наименований

## Решение
Изменена логика парсинга с "поколоночной" на **позиционную (построчную)**.

### Ключевые изменения в EXTRACT_SCRIPT

1. **Позиционный подход**: Парсер теперь обрабатывает данные по позициям, где каждая позиция начинается с строки вида "N. НАИМЕНОВАНИЕ" (например, "1. КЕТОРОЛАК", "6. ГАБАПЕНТИН").

2. **Группировка строк**: 
   - Основная строка позиции определяется по паттерну `^\s*(\d+)\.\s+`
   - Детальные строки (скрытые) идентифицируются по ID вида `ktru-tr-*`
   - Все детальные строки группируются с соответствующей основной позицией

3. **Обработка вложенных таблиц**: Для каждой позиции отдельно извлекаются данные из вложенных таблиц с информацией о торговых наименованиях, РУ, формах выпуска и дозировках.

4. **Слияние данных**: Данные из основной строки (количество, цена, сумма, ОКПД2, страна) сливаются с данными из детальных строк (торговое наименование, РУ, форма выпуска, дозировка, информация о производителе).

### Преимущества нового подхода

- **Исключено задвоение**: Каждая позиция обрабатывается один раз
- **Корректное связывание**: Данные из разных строк одной позиции правильно объединяются
- **Устойчивость к сдвигам**: Поскольку парсинг идёт от структуры документа, а не от фиксированных индексов столбцов
- **Обработка повторяющихся наименований**: Позиции различаются по номеру и контексту, а не только по наименованию
