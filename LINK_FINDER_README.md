# Модуль поиска ссылок для парсинга ЕИС (link_finder.py)

## Описание
Модуль предназначен для автоматического поиска ссылок на контракты в ЕИС (zakupki.gov.ru) 
с последующим экспортом их для дальнейшей обработки основным парсером.

## Отличия от основного парсера
- **Не выполняет парсинг данных контрактов** - только поиск ссылок
- **Выводит список целевых ссылок** в формате `payment-info-and-target-of-order.html?reestrNumber={NUMBER}`
- **Упрощенный интерфейс** - без таблицы результатов парсинга
- **Экспорт ссылок** в текстовый файл или буфер обмена

## Требования
- Python 3.9+
- PyQt5
- Selenium WebDriver
- ChromeDriver (устанавливается автоматически через webdriver-manager)
- pandas (для загрузки справочников МНН)
- requests
- openpyxl

## Установка зависимостей
```bash
pip install PyQt5 selenium webdriver-manager pandas requests openpyxl
```


## Параметры поиска

### Обязательные параметры
1. **Поисковый запрос (МНН)** - Международное непатентованное наименование лекарственного средства
   - Пример: "АЗИТРОМИЦИН"
   - Поддержка автодополнения из загруженного справочника

2. **Дата с** - Начальная дата размещения контракта
   - Формат: ДД.ММ.ГГГГ
   - По умолчанию: текущая дата минус 3 месяца

3. **Дата по** - Конечная дата размещения контракта
   - Формат: ДД.ММ.ГГГГ
   - По умолчанию: текущая дата

4. **Максимальное количество контрактов** - Лимит обрабатываемых карточек
   - По умолчанию: 20

### Опциональные параметры
1. **Искать только в Москве и МО** - Фильтр по кодам регионов: 77000000000,50000000000
   - Взаимно исключает параметр "Только РОСУНИМЕД"

2. **Искать только в РОСУНИМЕДЕ** - Фильтр по заказчику: Российский университет медицины
   - Взаимно исключает параметр "Только Москва и МО"

3. **База МНН** - Файл .xlsx со справочником для автодополнения
   - Лист должен начинаться с "esklp_smnn"

## Интерфейс

### Элементы управления
- **Поисковый запрос (МНН)** - QComboBox с поддержкой ручного ввода и автодополнения
- **Дата с / Дата по** - QDateEdit с календарем
- **Макс. контрактов** - QLineEdit для ввода числа
- **Чекбоксы фильтров** - mutually exclusive переключатели
- **Кнопка "Загрузить базу МНН"** - загрузка справочника для автодополнения
- **Кнопка "Найти закупки"** - запуск процесса поиска
- **Кнопка "Экспорт ссылок"** - сохранение найденных ссылок в файл
- **Кнопка "Копировать все"** - копирование всех ссылок в буфер обмена

### Отображение результатов
- **Список найденных ссылок** - QListWidget с возможностью двойного клика для открытия в браузере
- **Мини-лог** - компактное отображение процесса выполнения
- **Полный лог** - отдельное диалоговое окно (меню "Вид" → "Показать лог")
- **Статистика** - общее количество найденных ссылок

## Алгоритм работы

### Шаг 1. Формирование параметров запроса
- Сбор всех входных параметров из интерфейса
- Валидация дат и числовых параметров
- Проверка взаимного исключения региональных фильтров

### Шаг 2. Построение URL поиска ЕИС
Базовый URL: `https://zakupki.gov.ru/epz/contract/search/results.html`

Обязательные параметры:
- searchString = {SEARCH_TERM}
- morphology = on
- search-filter = Дате+размещения
- fz44 = on
- contractStageList_1 = on
- contractStageData = 1
- contractDateFrom = {DATE_FROM}
- contractDateTo = {DATE_TO}
- sortBy = UPDATE_DATE
- pageNumber = 1
- sortDirection = false
- recordsPerPage = _10
- showLotsInfoHidden = false
- strictEqual = true

Условные параметры:
- При выборе "Только Москва и МО": customerPlace, customerPlaceCodes
- При выборе "Только РОСУНИМЕД": customerIdOrg

### Шаг 3. Инициализация WebDriver
- Запуск Chrome в headless-режиме
- Отключение признаков автоматизации
- Настройка user-agent

### Шаг 4. Загрузка страниц результатов
- Выполнение GET-запроса к сформированному URL
- Повторные попытки при ошибках (до 3 раз)
- Определение общего числа страниц через пагинатор

### Шаг 5. Извлечение ссылок
Для каждой страницы:
- Извлечение всех ссылок с атрибутом href
- Фильтрация: оставить только содержащие "contract/contractCard/common-info.html"
- Уникализация ссылок
- Преобразование в целевой формат: `payment-info-and-target-of-order.html?reestrNumber={NUMBER}`

### Шаг 6. Вывод результатов
- Отображение списка ссылок в интерфейсе
- Возможность экспорта в текстовый файл
- Копирование в буфер обмена
- Открытие ссылки в браузере по двойному клику

## Обработка ошибок

### CAPTCHA
При обнаружении CAPTCHA:
- Вывод уведомления в лог
- Продолжение работы с задержкой
- Рекомендуется ручное прохождение CAPTCHA в браузере

### Ошибки загрузки страницы
- Автоматические повторные попытки (до 3 раз)
- Интервал между попытками: 3 секунды
- Логирование всех ошибок

### Потеря соединения
- Грациозное завершение работы
- Сохранение найденных ссылок
- Очистка ресурсов WebDriver в блоке finally

## Логирование
Все события записываются в файл `parser_log.txt`:
- Время события
- Уровень логирования (INFO, ERROR, DEBUG)
- Описание события

Пример:
```
2025-01-15 10:30:45,123 - INFO - Запрос: https://zakupki.gov.ru/epz/contract/search/results.html?...
2025-01-15 10:30:50,456 - INFO - Страница 1/5
2025-01-15 10:30:55,789 - INFO - Найдено уникальных ссылок на странице: 10
```

## Интеграция с основным парсером
Найденные ссылки можно использовать в основном парсере следующим образом:

1. Экспортировать ссылки в файл через кнопку "Экспорт ссылок"
2. В основном парсере реализовать загрузку ссылок из файла
3. Обработать каждую ссылку для извлечения данных о контракте

## Пример использования

1. Запустить модуль: `python link_finder.py`
2. Ввести поисковый запрос: "АЗИТРОМИЦИН"
3. Выбрать даты: с 01.01.2025 по 31.01.2025
4. Установить макс. контрактов: 20
5. При необходимости выбрать фильтры (Москва/МО или РОСУНИМЕД)
6. Нажать "Найти закупки"
7. Дождаться завершения поиска
8. Экспортировать ссылки или скопировать в буфер обмена
9. Использовать ссылки в основном парсере
