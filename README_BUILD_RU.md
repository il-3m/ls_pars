# 🚀 Unified Parser - Готовый проект для сборки EXE

## ✅ Что исправлено в этой версии

### Проблема
Ранее EXE файл выдавал ошибку:
```
Executable doesn't exist at C:\Users\...\Temp\_MEI17162\playwright\driver\chromium-...\chrome.exe
```

Это происходило потому, что PyInstaller в режиме one-file распаковывает всё во временную папку `_MEIxxxxx`, но Playwright искал браузеры там, а не в постоянном кэше.

### Решение
1. **Используем постоянный кэш браузеров**: `PLAYWRIGHT_BROWSERS_PATH` теперь указывает на `%USERPROFILE%\.cache\ms-playwright`
2. **Автоустановка при первом запуске**: Если браузеры не найдены, Playwright автоматически скачает их (~150 МБ)
3. **Понятные сообщения об ошибках**: Чёткие инструкции, что делать

## 🔨 Как собрать EXE на Windows

### Требования для сборки:
- Windows 10/11
- Python 3.8, 3.9, 3.10, 3.11 или 3.12 (не 3.13!)
- Стабильный интернет

### Пошаговая инструкция:

#### Шаг 1: Подготовьте папку
Скопируйте все 5 файлов в одну папку:
```
unified_parser.py
eis_parser.py
link_finder.py
requirements.txt
build_exe.bat
```

#### Шаг 2: Запустите сборку
Дважды кликните на `build_exe.bat` или запустите через командную строку:
```cmd
build_exe.bat
```

#### Шаг 3: Дождитесь завершения (5-15 минут)
Скрипт автоматически:
1. ✅ Проверит наличие Python
2. ✅ Установит зависимости (PyQt5, selenium, pandas, playwright и др.)
3. ✅ Скачает браузеры Playwright (~150 МБ)
4. ✅ Соберёт EXE файл

#### Шаг 4: Найдите готовый EXE
Готовый файл будет в папке:
```
export\Unified_Parser.exe
```

## 🎯 Использование на целевом компьютере

### Требования:
✅ **Нужно:**
- Windows 10/11
- Google Chrome (для первой части парсера - поиск ссылок)
- Интернет (для первого запуска + парсинга)

❌ **НЕ нужно:**
- Python
- Ручная установка Playwright
- Какие-либо настройки

### Первый запуск (1-2 минуты):
1. Запустите `Unified_Parser.exe`
2. Playwright обнаружит отсутствие браузеров
3. Автоматически скачает Chromium (~150 МБ)
4. Сохранит в `%USERPROFILE%\.cache\ms-playwright`
5. Продолжит работу парсера

**Важно:** В интерфейсе может появиться сообщение о загрузке браузеров - это нормально!

### Последующие запуски:
- Браузеры уже в кэше
- Запуск мгновенный
- Никаких дополнительных загрузок

## ⚠️ Возможные проблемы и решения

### 1. Ошибка при сборке: "Failed to build pandas"
**Причина:** Python 3.13 не имеет готовых бинарных пакетов

**Решение:**
```cmd
# Вариант 1: Установите Python 3.11 или 3.12
# Скачайте с https://www.python.org/downloads/

# Вариант 2: Попробуйте установить только бинарные пакеты
pip install -r requirements.txt --only-binary :all:
```

### 2. Ошибка при первом запуске EXE: "Executable doesn't exist"
**Это нормально!** Просто подождите 1-2 минуты пока Playwright скачивает браузеры.

**Если не помогает:**
- Проверьте подключение к интернету
- Отключите антивирус/фаервол на время загрузки
- Запустите EXE от имени администратора

### 3. EXE файл блокируется антивирусом
**Причина:** Ложное срабатывание на PyInstaller

**Решение:**
- Добавьте исключение в антивирус
- Или подпишите EXE цифровым сертификатом

### 4. Долгая сборка (>30 минут)
**Причина:** Медленный интернет или слабый компьютер

**Решение:**
- Проверьте скорость интернета
- Закройте другие программы
- Попробуйте использовать зеркало pip:
```cmd
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 📊 Размеры файлов

- **Исходный код**: ~200 КБ
- **EXE файл**: ~50-80 МБ (сжатый PyInstaller)
- **Загрузка при первом запуске**: ~150 МБ (браузер Chromium)
- **Итого на диске**: ~200-250 МБ

## 🔧 Технические детали

### Почему браузеры не встроены в EXE?

1. **Размер**: EXE стал бы 400+ МБ (медленно распространять)
2. **Проблемы с temp-папкой**: PyInstaller one-file распаковывает во временную папку
3. **Обновления**: Раздельный кэш позволяет Playwright обновлять браузеры независимо

### Как это работает:

```
Первый запуск:
EXE → Проверка %USERPROFILE%\.cache\ms-playwright → Не найдено →
Загрузка Chromium → Сохранение в кэш → Парсинг

Второй запуск:
EXE → Проверка %USERPROFILE%\.cache\ms-playwright → Найдено! →
Парсинг сразу
```

## 📝 Логи и отладка

Если что-то пошло не так:

1. **Проверьте лог-файл**: `unified_parser_log.txt` (создаётся в той же папке)
2. **Запустите из командной строки**: Чтобы видеть вывод в консоли
   ```cmd
   Unified_Parser.exe
   ```
3. **Включите режим с окном**: При сборке измените `--windowed` на `--console` в `build_exe.bat`

## 🆘 Поддержка

Если проблемы не решаются:

1. Проверьте версию Python: `python --version` (должна быть 3.8-3.12)
2. Обновите pip: `python -m pip install --upgrade pip`
3. Попробуйте собрать заново после очистки:
   ```cmd
   rmdir /s /q build dist
   del *.spec
   build_exe.bat
   ```
