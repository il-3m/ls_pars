# Инструкция по использованию модуля поиска контрактов

## Консольный режим (автоматический поиск и парсинг)

### Базовый запуск
```bash
python eis_parser.py --search "АЗИТРОМИЦИН"
```
Поиск контрактов с лекарством "АЗИТРОМИЦИН" за последние 3 месяца (макс. 20 контрактов).

### Полный набор параметров
```bash
python eis_parser.py --search "ПАРАЦЕТАМОЛ" \
    --date-from "01.01.2024" \
    --date-to "31.03.2024" \
    --max-contracts 10 \
    --out-csv "export/paracetamol.csv" \
    --out-xlsx "export/paracetamol.xlsx"
```

### Фильтры по регионам
**Только Москва и Московская область:**
```bash
python eis_parser.py --search "ИНСУЛИН" --moscow-only
```

**Только РОСУНИМЕД:**
```bash
python eis_parser.py --search "ГЕПАРИН" --rosunimed-only
```

### Все доступные аргументы

| Аргумент | Описание | По умолчанию |
|----------|----------|--------------|
| `--search` | Поисковый запрос (МНН) | Обязательно |
| `--date-from` | Дата с (ДД.ММ.ГГГГ) | -90 дней |
| `--date-to` | Дата по (ДД.ММ.ГГГГ) | Сегодня |
| `--max-contracts` | Макс. количество контрактов | 20 |
| `--moscow-only` | Только Москва и МО | False |
| `--rosunimed-only` | Только РОСУНИМЕД | False |
| `--out-csv` | Путь к CSV файлу | export/result.csv |
| `--out-xlsx` | Путь к XLSX файлу | export/result.xlsx |

## Примеры использования

### Пример 1: Базовый поиск
```bash
python eis_parser.py --search "МЕТАМИЗОЛ НАТРИЯ"
```

### Пример 2: Поиск с фильтром по дате
```bash
python eis_parser.py --search "ДИКЛОФЕНАК" \
    --date-from "01.02.2024" \
    --date-to "28.02.2024"
```

### Пример 3: Поиск в Москве с ограничением
```bash
python eis_parser.py --search "ПРЕДНИЗОЛОН" \
    --moscow-only \
    --max-contracts 5
```

### Пример 4: Поиск с экспортом в Excel
```bash
python eis_parser.py --search "АМПИЦИЛЛИН" \
    --out-csv "results/ampicillin.csv" \
    --out-xlsx "results/ampicillin.xlsx"
```

## Принцип работы

1. **Поиск ссылок**: Парсер автоматически находит страницы контрактов на zakupki.gov.ru по заданным параметрам
2. **Извлечение данных**: Для каждой найденной ссылки вызывается основной алгоритм парсинга
3. **Накопление результатов**: Данные последовательно добавляются в общую таблицу
4. **Экспорт**: Результаты сохраняются в CSV и опционально в XLSX

## Выходные данные

Результаты содержат 16 колонок:
1. Наименование
2. Страна
3. КТРУ/ОКПД2
4. Тип объекта
5. Количество
6. Цена за ед.
7. Сумма с НДС
8. Заказчик
9. № контракта
10. № реестра
11. Лек. форма
12. Дозировка
13. Дата контракта
14. Торговое наименование
15. Номер РУ
16. Ссылка

## Обработка CAPTCHA

При обнаружении CAPTCHA парсер пропускает текущую ссылку и продолжает обработку следующих. Для полного обхода требуется ручное вмешательство пользователя.

## Логи

Все события записываются в файл `parser_log.txt`.
