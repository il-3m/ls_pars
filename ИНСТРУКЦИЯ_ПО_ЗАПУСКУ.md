# 🚀 Инструкция по запуску парсера лекарств ЕИС

## ⚡ Быстрый старт (Windows)

### Способ 1: Двойной клик (рекомендуется)

1. **Дважды кликните** на файл `run_parser_gui.bat`
2. Дождитесь открытия окна парсера
3. Заполните параметры поиска
4. Нажмите кнопку **"🚀 Запустить парсинг"**

### Способ 2: Через командную строку

```bash
python parser_launcher.py
```

---

## 📋 Как пользоваться интерфейсом

### 1️⃣ Параметры поиска (обязательно)

| Поле | Описание | Пример |
|------|----------|--------|
| **Поисковый запрос (МНН)** | Международное непатентованное наименование лекарства | `АЗИТРОМИЦИН`, `ПАРАЦЕТАМОЛ`, `ИНСУЛИН` |
| **Период поиска** | Диапазон дат для поиска контрактов | По умолчанию: последние 3 месяца |
| **Макс. контрактов** | Сколько контрактов обработать за один запуск | По умолчанию: 20 (можно от 1 до 1000) |

### 2️⃣ Фильтры (опционально)

| Фильтр | Описание |
|--------|----------|
| ☑️ **Искать только в Москве и МО** | Поиск контрактов только с заказчиками из Москвы (77) и Московской области (50) |
| ☑️ **Искать только в РОСУНИМЕД** | Поиск контрактов только Российского университета медицины |

> ⚠️ **Важно:** Эти два фильтра взаимно исключают друг друга. Можно выбрать только один.

### 3️⃣ Запуск парсинга

1. Нажмите кнопку **"🚀 Запустить парсинг"**
2. В окне журнала будет отображаться процесс выполнения
3. Дождитесь сообщения **"✅ Парсинг завершён успешно!"**

### 4️⃣ Результаты

После завершения парсинга результаты сохраняются в папке `export/`:

- `result.csv` — таблица в формате CSV
- `result.xlsx` — таблица в формате Excel с форматированием

---

## 📊 Структура результатов

Парсер собирает следующие данные по каждому контракту:

1. Наименование (МНН)
2. Страна происхождения
3. КТРУ/ОКПД2
4. Тип объекта закупки
5. Количество/объём
6. Цена за единицу
7. Сумма с НДС
8. Наименование заказчика
9. Номер контракта
10. Номер реестровой записи
11. Лекарственная форма
12. Дозировка
13. Дата заключения контракта
14. Торговое наименование
15. Номер регистрационного удостоверения
16. Ссылка на страницу контракта

## 🔧 Требования к системе

### Обязательные компоненты

- **Python 3.9** или выше
- **Google Chrome** (для работы Selenium)
- **Интернет-соединение**

### Автоматически устанавливаемые зависимости

При первом запуске скрипт автоматически установит:

```bash
pip install PyQt5 selenium webdriver-manager pandas openpyxl
```

Или установите вручную:

```bash
pip install PyQt5 selenium webdriver-manager pandas openpyxl requests playwright
playwright install
```

## ❓ Частые вопросы

### Вопрос: Окно не открывается
**Решение:**
1. Убедитесь, что установлен Python 3.9+
2. Проверьте, что Python добавлен в PATH
3. Запустите от имени администратора
4. Проверьте антивирус (может блокировать)

### Вопрос: Появляется CAPTCHA
**Решение:**
1. Парсер обнаружит CAPTCHA и покажет уведомление
2. Нажмите "Да" для продолжения
3. Вручную решите CAPTCHA в браузере (у вас будет 10 секунд)
4. Парсер продолжит работу автоматически

### Вопрос: Найдено 0 контрактов
**Возможные причины:**
- Слишком узкий период поиска
- Редкое МНН (попробуйте более общее название)
- Включены фильтры, которые исключают все результаты

**Решение:** Расширьте период поиска или отключите фильтры

### Вопрос: Парсинг зависает
**Решение:**
1. Нажмите кнопку "⏹️ Остановить"
2. Проверьте интернет-соединение
3. Попробуйте уменьшить количество контрактов
4. Перезапустите парсер

## 🛑 Остановка парсинга

Для остановки выполнения:
1. Нажмите кнопку **"⏹️ Остановить"**
2. Подтвердите остановку в диалоговом окне

---

## 💡 Советы по эффективному использованию

1. **Начинайте с малого**: Укажите 5-10 контрактов для тестирования
2. **Проверяйте МНН**: Используйте официальные названия из справочника
3. **Оптимальный период**: 1-3 месяца дают лучший баланс скорости и полноты
4. **Фильтры**: Используйте только если точно знаете, что нужно
5. **Сохраняйте логи**: При ошибках сохраняйте журнал для анализа

---

## 📞 Поддержка

При возникновении проблем:
1. Проверьте файл `parser_log.txt` в папке с программой
2. Сохраните скриншот ошибки
3. Проверьте версию Python: `python --version`
