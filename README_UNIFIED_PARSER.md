# Универсальный парсер ЕИС (Unified Parser)

Этот скрипт объединяет две части парсера в единый рабочий процесс:
1. **Поиск ссылок** (из `link_finder.py`) - находит контракты по заданным критериям
2. **Парсинг данных** (из `eis_parser.py`) - извлекает данные из страниц контрактов

## Возможности

- **Единый интерфейс**: Не нужно запускать два разных скрипта
- **Автоматический цикл**: Нашел ссылку → обработал → добавил в таблицу → следующая ссылка
- **Накопление данных**: Все распарсенные строки собираются в одну итоговую таблицу
- **Два режима работы**: GUI (графический интерфейс) и консольный режим
- **Гибкие настройки**: Можно настроить таймауты, фильтры, форматы экспорта

## Запуск

### Графический интерфейс (GUI)

```bash
python unified_parser.py --gui
```

Или просто без аргументов (GUI запускается по умолчанию):

```bash
python unified_parser.py
```

### Консольный режим

```bash
python unified_parser.py --search "АЗИТРОМИЦИН" --max-contracts 10 --out-csv result.csv
```

#### Параметры консольного режима:

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--search` | Поисковый запрос (МНН) | `"АЗИТРОМИЦИН"` |
| `--date-from` | Дата с (YYYY-MM-DD) | `2024-01-01` |
| `--date-to` | Дата по (YYYY-MM-DD) | `2024-12-31` |
| `--max-contracts` | Максимум контрактов | `20` |
| `--moscow-only` | Только Москва и МО | (флаг) |
| `--rosunimed-only` | Только Росунимед | (флаг) |
| `--archive-dir` | Папка для архива | `archive` |
| `--out-csv` | Выходной CSV файл | `export/result.csv` |
| `--out-xlsx` | Выходной XLSX файл | `export/result.xlsx` |
| `--headed` | Браузер с окном (не headless) | (флаг) |
| `--timeout-ms` | Таймаут загрузки (мс) | `90000` |
| `--expand-rounds` | Раунды раскрытия элементов | `5` |

### Примеры команд

```bash
# Поиск по МНН за последние 3 месяца
python unified_parser.py --search "ИБУПРОФЕН" --max-contracts 50

# Только Москва, экспорт в Excel
python unified_parser.py --search "ПАРАЦЕТАМОЛ" --moscow-only --out-xlsx result.xlsx

# Полный цикл с настройками таймаутов
python unified_parser.py --search "МЕТАМИЗОЛ" --timeout-ms 120000 --expand-rounds 7
```

## Как это работает

### Этап 1: Поиск ссылок
1. Открывается страница поиска ЕИС
2. Формируется запрос с параметрами (МНН, даты, регион)
3. Проходятся все страницы результатов
4. Извлекаются ссылки на карточки контрактов
5. Ссылки преобразуются в формат для парсинга

### Этап 2: Парсинг данных
1. Для каждой найденной ссылки:
   - Открывается страница контракта в Playwright
   - Раскрываются все элементы "Объекты закупки"
   - Извлекаются данные таблицы (наименование, цена, производитель и т.д.)
   - Данные добавляются в общий список
2. После обработки всех ссылок:
   - Данные экспортируются в CSV
   - Опционально создается XLSX файл

### Структура выходных данных

Таблица содержит следующие поля:
- Наименование
- Категории ЛС
- ОКПД2
- Страна происхождения
- МНН (ГРЛС)
- Торговое наименование
- Регистрационное удостоверение (РУ)
- Форма выпуска
- Дозировка
- Количество в потреб. единице
- Цена за единицу (₽)
- Сумма (₽)
- НДС
- Держатель РУ
- Производитель
- Страна производителя
- Вид первичной упаковки
- И другие технические поля

## Требования

- Python 3.8+
- PyQt5 (для GUI)
- Selenium + webdriver-manager (для поиска ссылок)
- Playwright (для парсинга)
- pandas, openpyxl (опционально, для XLSX)

### Установка зависимостей

```bash
pip install PyQt5 selenium webdriver-manager playwright pandas openpyxl
playwright install chromium
```

## Логирование

Все действия записываются в файл `unified_parser_log.txt`:
- Время начала/окончания
- Найденные ссылки
- Ошибки парсинга
- Статистика по строкам

## Решение проблем

### Ошибка "libtk8.6.so not found"
Это ошибка tkinter, не влияет на работу GUI (используется PyQt5). 
Для консольного режима можно игнорировать.

### CAPTCHA при поиске
При обнаружении CAPTCHA парсер делает паузу. Рекомендуется:
- Уменьшить количество контрактов за один проход
- Добавить задержки между запросами
- Использовать прокси

### Нет данных в таблице
Возможные причины:
- Контракт не содержит раздел "Объекты закупки"
- Изменилась структура сайта ЕИС
- Требуется больше времени на загрузку (увеличьте `--timeout-ms`)
